# Хранилище знаний — инструкция для агента

## Общие правила
- Язык вики: русский.
- Сырые источники в `raw/` ИММУТАБЕЛЬНЫ — только чтение.
- Все страницы в `wiki/` создаёт и поддерживает агент.
- Frontmatter у каждой wiki-страницы: `date`, `source_type`, `status`, `tags`.

## Понимание пользователя
Три обязательных слоя:
- `wiki/raw-thoughts/` — дословный голос пользователя. НИКОГДА не редактируй.
- `wiki/insights/` — структурированные мысли с прямыми цитатами. Минимум твоих слов, максимум цитат пользователя.
- `wiki/thinking/` — эволюция мысли. Дельты с датами, старое НЕ перезаписывается.

## Эволюция мысли — стиль пользователя
{{СГЕНЕРИРОВАТЬ: один из четырёх вариантов в зависимости от выбора пользователя}}

Варианты (включи ТОЛЬКО один):
- [тема]: Страницы thinking группируются вокруг тем. Заголовок = тема, секция «Текущее состояние мысли» = последняя формулировка понимания темы.
- [вопрос]: Страницы thinking группируются вокруг вопросов. Заголовок = вопрос, секция «Текущий ответ» = к чему пользователь пришёл сейчас.
- [проект]: Страницы thinking привязаны к проектам. Заголовок = проект, секция «Текущее видение» = как пользователь сейчас видит проект.
- [смешанно]: При создании каждой thinking-страницы определи ось (тема/вопрос/проект) и укажи в frontmatter `thinking_axis`.

## Связь raw-thoughts <-> insights
У каждой реакции пользователя есть пара: дословный файл в raw-thoughts + причёсанный в insights.
Они ссылаются друг на друга через frontmatter и inline-ссылки.

## Шаблоны страниц
При создании новой страницы используй шаблон из `wiki/_templates/`.
Заполни все поля frontmatter. Замени плейсхолдеры `{{...}}` на реальные данные.
Формат ссылок: `[[wiki/путь/название]]`.

## ОПЕРАЦИЯ: INGEST
Когда пользователь говорит «загрузи источник» или кладёт файл в `raw/`:
1. Прочитай источник целиком.
2. Покажи 3-5 ключевых мыслей СВОИМИ СЛОВАМИ и спроси: «что откликнулось?». ЖДИ ответ.
3. После ответа:
   - Создай/обнови страницу по теме в соответствующей папке wiki.
   - Реплики пользователя сохрани в `wiki/raw-thoughts/` — ДОСЛОВНО.
   - На основе raw-thought создай причёсанный инсайт в `wiki/insights/`.
   - Проставь двусторонние ссылки.
   - Если пользователь указал связи — используй ЕГО логику связей.
4. Обнови `index.md` и `log.md`.
5. Покажи список затронутых файлов.

## ОПЕРАЦИЯ: QUERY
Когда пользователь задаёт вопрос по вики:
1. Прочитай `index.md`, найди релевантные страницы.
2. Ответь с цитатами и ссылками.
3. Спроси: «сохранить ответ как страницу?».
4. Запиши в `log.md`.

## ОПЕРАЦИЯ: LINT
Когда пользователь говорит «сделай lint»:
1. Пройди по всем страницам в `wiki/`.
2. Найди: противоречия, сироты, упоминания без страниц, заброшенные практики.
3. Предложи вопросы для исследования.
4. НИЧЕГО не правь — только отчёт.

## Чего НЕ делать
- Не редактируй `wiki/raw-thoughts/` — только дословно.
- Не заменяй голос пользователя своим пересказом в insights.
- Не сливай итерации мыслей в «финальную версию».
- Не удаляй `raw/` и `wiki/raw-thoughts/`.
